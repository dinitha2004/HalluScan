## 1. "fusion" යනු කුමක්ද? (සරල අදහස)

මොඩලය පිළිතුරක් දෙන විට, detector තුනක් එය බලා එක් අගයක් (number) බැගින් දෙයි:

| Head | එය බලන්නේ | ප්‍රතිදානය (වැඩි නම් = වැරදි තොරතුරක් වීමේ ඉඩ වැඩියි) |
|---|---|---|
| **SEP** | මොඩලයේ අභ්‍යන්තර "මනස් ඡායාරූපය" අවිනිශ්චිත/ගොතන ලද ලෙස පෙනේද? | `sep_entropy` (0–1) |
| **HalluShift** | පිළිතුර දෙන විට layers අතර අසාමාන්‍ය ලෙස "shift" වුණාද? | `hallushift` (0–1) |
| **TSV** | පිළිතුර "බොරුවක" ඉරියව්වටද නැතිනම් "සත්‍යයක" ඉරියව්වටද නැඹුරුද? | `tsv_margin` (≈ −0.3 … +0.3) |

එක් detector එකක් පමණක් විශ්වාසදායක නැත. **fusion** එක යනු දත්තවලින් *ඉගෙන ගත්* කුඩා විනිශ්චයකරුවෙකි — එක් එක් detector එකට කොතරම් විශ්වාසය තැබිය යුතුද යන්න ඉගෙන ගෙන, අගයන් තුන **අවසන් එක් සම්භාවිතාවකට** (0–1) ඒකාබද්ධ කරයි.

## 3. Head තුනෙන් fusion එක සාදන්නේ කෙසේද

`tools/train_claim_fusion.py` මඟින්:

1. **පුහුණු** පේළි ගන්න: එක් එක් පේළියට `[sep_entropy, hallushift, tsv_margin]` සහ සත්‍ය/අසත්‍ය ලේබලයක් ඇත.
2. තීරු තුන **standardize** කරන්න (සාමාන්‍යය අඩු කර, විසිරීමෙන් බෙදන්න) — SEP හි 0–1 පරාසය සහ TSV හි ±0.3 පරාසය සැසඳිය හැකි බවට පත් කිරීමට.
3. **logistic regression** (සරල බර‑එකතු කිරීමේ සූත්‍රයක්) ගලපන්න. මෙය එක් head එකකට **බරක්** (weight) සහ එක bias එකක් ඉගෙන ගනී.
4. **එළිපත්ත (thresholds) දෙකක් කැලිබ්‍රේට්** කරන්න (පුහුණු භාගයෙන් පමණක්): `t_high` = හොඳම F1 දෙන කැපුම, සහ `t_med = 0.6 × t_high`. මේවා සම්භාවිතාව Reliable / Uncertain / Likely‑Hallucinated band තුනට හරවයි.
5. `{බර, bias, scaler, feature ලැයිස්තුව}` `fusion_claim_<tag>.pkl` එකටත්, thresholds `..._thresholds.json` එකටත් save කරයි.

## 4. fusion ගණිතය (බර කරන්නේ කුමක්ද)

එක් පිළිතුරකට (හෝ එක් වාක්‍යයකට) fusion එක පියවර තුනක් කරයි:

```
පියවර 1 — එක් එක් ලකුණ standardize:   z_i = (x_i − mean_i) / scale_i
පියවර 2 — බර සහිත එකතුව + bias:       logit = (w_sep·z_sep) + (w_hs·z_hs) + (w_tsv·z_tsv) + bias
පියවර 3 — සම්භාවිතාවකට හකුළුවන්න:      fused = 1 / (1 + e^(−logit))     # sigmoid → 0..1
```

- `mean_i`, `scale_i` පුහුණු දත්තවලින් එයි (**scaler** එක).
- `w_i` (**coefficients/බර**) — එක් head එකක් කොතරම් වැදගත්ද කියයි; බර වැඩි නම් ඒ head එක ආධිපත්‍ය දරයි.
- `bias` (**intercept**) — සියලු standardized ලකුණු 0 වන විට මූලික අගයයි.
- `fused` යනු අවසන් **වැරදි තොරතුරක් වීමේ සම්භාවිතාවයි**.

> code එක මෙම sigmoid එක save කළ බරවලින් අතින් ගණනය කරයි (`hallking/fusion.py` හි `FusionModel._proba1`), එම නිසා save කළ ගොනුව library අනුවාද හරහා වැඩ කරයි.

## 5. 1B මොඩලය — `fusion_claim_l1b.pkl`

```
feature_cols : ['sep_entropy', 'hallushift', 'tsv_margin']    (head 3ම)
scaler.mean_ : [0.4768, 0.4885,  0.0148]
scaler.scale_: [0.4671, 0.2461,  0.1026]
coef_ (w)    : [4.5894, 0.2780,  0.2890]    ← SEP ආධිපත්‍ය (4.59 ≫ 0.28, 0.29)
intercept_   : [-0.2442]
thresholds   : t_med = 0.155,  t_high = 0.258
```

**සරල අර්ථය:** head තුනම ඇතුළත් වුවද, ගණිතය බොහෝ දුරට **SEP** මත රඳා පවතී (බර 4.59 එදිරිව අනෙක් දෙකට ~0.28). එම නිසා 1B හි fused ලකුණ බොහෝ දුරට SEP වන අතර, HalluShift සහ TSV කුඩා බලපෑමක් දෙයි.

**සූත්‍රය:**
```
z_sep = (sep_entropy − 0.4768)/0.4671
z_hs  = (hallushift  − 0.4885)/0.2461
z_tsv = (tsv_margin  − 0.0148)/0.1026
fused = sigmoid( 4.5894·z_sep + 0.2780·z_hs + 0.2890·z_tsv − 0.2442 )
```

**උදාහරණ (1B):**

| sep_entropy | hallushift | tsv_margin | logit | **fused** | tier |
|---|---|---|---|---|---|
| 0.00 | 0.154 | −0.167 | −5.82 | **0.003** | Reliable ✅ (සැබෑ "ප්‍රංශයේ අගනුවර?" පිළිතුර) |
| 0.95 | 0.70 | +0.20 | +5.17 | **0.994** | Likely Hallucinated 🚨 |

SEP කොටස ප්‍රතිඵලය මෙහෙයවන බව පෙනේ: අඩු `sep_entropy` (0.00) පමණක් ලකුණ 0.155 කැපුමට වඩා බොහෝ පහළට තල්ලු කරයි → Reliable.

## 6. 8B මොඩලය — `fusion_claim_8bv2.pkl` (නව, notebook 15 හි)
